# Amit's Daily Game Factory 🎮

אוטומציה שיוצרת בכל יום משחק דפדפן גדול יחסית, בודקת אותו, מוסיפה אותו לגלריה, מפרסמת ב-Netlify ושולחת את הקישור במייל.

## מה כבר יש בפנים

- סוכן תכנון שממציא משחק מקורי ולא חוזר על המשחקים האחרונים.
- סוכן בנייה שיוצר משחק Canvas מלא בקובץ HTML אחד.
- בדיקות אוטומטיות: גודל מינימלי, Canvas, לולאת משחק, שמירה, מגע, עצירה ובדיקת תחביר JavaScript.
- עד שני ניסיונות תיקון אוטומטיים אם המשחק לא עובר בדיקה.
- לפחות 8 שלבים/גלים/משימות, שדרוגים, שמירה, מסכי ניצחון והפסד, מובייל ומחשב.
- גלריה ששומרת גם את המשחקים הקודמים.
- העלאה אוטומטית ל-Netlify.
- שליחת קישור אוטומטית דרך Gmail.
- משחק הדגמה מוכן: **Pulse Orbit**.

## חשוב לפני שמתחילים

אל תדביק מפתחות API, סיסמאות או טוקנים בצ'אט, בקוד או בקובץ שמעלים ל-GitHub. מכניסים אותם רק ל-**GitHub Secrets**. בגלל שמדובר בחשבונות ושירותים חיצוניים, כדאי שהורה יעזור ביצירת החשבונות ובהגדרת הסודות.

## הקמה — פעם אחת בלבד

### 1. העלה את התיקייה ל-GitHub

1. צור Repository חדש ב-GitHub.
2. חלץ את קובץ ה-ZIP.
3. פתח את התיקייה `UPLOAD_TO_GITHUB`, בחר את **כל התוכן שבתוכה**, כולל `.github`, וגרור לעמוד ההעלאה.
4. ודא שהקובץ `.github/workflows/daily-game.yml` קיים באתר GitHub.

### 2. צור אתר ריק ב-Netlify

1. היכנס ל-Netlify וצור אתר חדש.
2. בפעם הראשונה אפשר להעלות ידנית את תיקיית `public` כדי לקבל כתובת כמו:
   `https://your-site-name.netlify.app`
3. העתק את **Site ID / Project ID** של האתר.
4. צור ב-Netlify **Personal Access Token**.

### 3. צור מפתח Gemini

צור מפתח API ב-Google AI Studio. המודל שמוגדר כברירת מחדל הוא `gemini-2.5-flash`. אפשר לשנות אותו בקובץ ה-Workflow בעתיד.

### 4. הגדר שליחת מייל

הדרך שמוגדרת בפרויקט היא Gmail App Password:

1. הפעל אימות דו-שלבי בחשבון Gmail ששולח את ההודעה.
2. צור App Password ייעודי לאוטומציה.
3. אל תשתמש בסיסמת Gmail הרגילה.

### 5. הוסף GitHub Secrets

ב-Repository פתח:

`Settings → Secrets and variables → Actions → New repository secret`

הוסף את הסודות הבאים:

| שם הסוד | מה להכניס |
|---|---|
| `GEMINI_API_KEY` | מפתח Gemini |
| `NETLIFY_AUTH_TOKEN` | הטוקן של Netlify |
| `NETLIFY_SITE_ID` | ה-Site ID / Project ID |
| `SITE_URL` | הכתובת המלאה של האתר, בלי `/` בסוף |
| `GMAIL_USER` | כתובת Gmail ששולחת |
| `GMAIL_APP_PASSWORD` | App Password, לא הסיסמה הרגילה |
| `EMAIL_TO` | הכתובת שאליה יגיע קישור המשחק |

### 6. הפעל בפעם הראשונה

1. ב-GitHub פתח `Actions`.
2. בחר `Daily Game Factory`.
3. לחץ `Run workflow`.
4. בסיום יגיע מייל עם קישור למשחק החדש.

לאחר מכן הריצה מתבצעת אוטומטית בכל יום לפי השעה שבקובץ ה-Workflow.

## שינוי שעת ההפעלה

בקובץ:

`.github/workflows/daily-game.yml`

שנה את השורה:

```yaml
- cron: "0 14 * * *"
```

GitHub משתמש בשעון UTC. כרגע זה 17:00 בישראל בזמן שעון קיץ ו-16:00 בזמן שעון חורף.

## תצוגה מקומית בטוחה במחשב Windows

אין בתיקייה קובץ `.env`, ואין צורך ליצור אחד כדי לראות את משחק ההדגמה.

לחץ פעמיים על:

`START_PREVIEW.bat`

הוא יפתח את הגלריה המקומית בכתובת `http://localhost:3000`. יצירת משחק חדש, העלאה ומייל מתבצעים בענן דרך GitHub Actions והסודות נשמרים רק ב-GitHub Secrets.

## Google Search

Netlify מפרסם את האתר באינטרנט ונותן קישור מיד. הפרויקט גם יוצר `robots.txt` ו-`sitemap.xml`. כדי לעזור לגוגל למצוא את האתר, הוסף את כתובת האתר פעם אחת ל-Google Search Console ושלח שם את `sitemap.xml`. אין דרך תקינה להבטיח שכל משחק רגיל יופיע בגוגל מיד; ה-Indexing API של Google מיועד בעיקר לדפי משרות ושידורים חיים, לא למשחקים רגילים.

## איך המפעל עובד

1. קורא את המשחקים הקודמים.
2. בוחר שילוב חדש של עולם ומכניקה.
3. Gemini מתכנן את המשחק.
4. Gemini כותב HTML מלא.
5. הסקריפט בודק את המשחק.
6. במקרה של תקלה, Gemini מקבל את השגיאות ומתקן.
7. המשחק נכנס ל-`public/games/`.
8. הגלריה, robots.txt וה-sitemap מתעדכנים.
9. GitHub שומר את המשחק ב-Repository.
10. Netlify מפרסם את כל הגלריה.
11. Gmail שולח את הקישור.

## פתרון תקלות מהיר

- **Missing required environment variable** — חסר Secret ב-GitHub.
- **Gemini returned 400/404** — בדוק את המפתח או החלף את `GEMINI_MODEL`.
- **Netlify deploy failed** — בדוק את הטוקן ואת Site ID.
- **Invalid login / Gmail** — צריך App Password, לא סיסמה רגילה.
- **המשחק לא נוצר** — ב-Actions נוצר קובץ Debug שאפשר להוריד למשך 7 ימים.
